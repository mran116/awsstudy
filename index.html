📋 How to add cards — format guide For you and for Claude ▶

When asking Claude to add new missed questions, paste the question and say "add this to my AWS quiz card." Claude should follow this exact format:

Question
Full exam scenario exactly as it appeared. Include all context — the company situation, the constraint, what is being asked. End with a clear question. Do not summarize or shorten it.
"A company needs to collect gigabytes of data per second from social media feeds and process it in real time. The solution must support reading records in batches and have built-in enhanced fan-out. Which AWS service should be used?"
Answer choices (A, B, C, D)
All four answer choices exactly as written. For questions with two correct answers, mark both (e.g. correct = "BC").
A — Amazon S3 with Athena   B — AWS Data Exchange   C — Amazon Kinesis Data Streams   D — Amazon MSK
✓ Correct answer explanation
WHY the correct answer is right in 2–3 sentences. Include the specific mechanism or feature that makes it correct — not just "this works."
"Kinesis Data Streams ingests data with millisecond latency. It supports batch reading via Lambda and has a built-in enhanced fan-out feature that gives each consumer dedicated throughput per shard."
✗ The trap
Why the most tempting wrong answer fails. Focus on the specific reason — missing feature, wrong use case, subtle distinction. This is the most important part for learning.
"Data Firehose is near-real-time (60s buffer minimum), not real-time, and does not support EC2 consumers or enhanced fan-out. MSK can stream in real time but has high admin overhead and no built-in enhanced fan-out."
Memorize this — key rule
One or two short sentences capturing the essential distinction. Use the format: "X = this. Y = that."
"KDS = real-time (ms), custom consumers, enhanced fan-out. Firehose = near real-time, managed delivery to S3/Redshift/OpenSearch only."

Cards auto-rate by whether the answer is correct. Missed → repeats next session. Correct → interval doubles: 1 day → 3 days → 7 days → 14 days. Use ↩ Review again to keep a card in rotation even after a correct answer.
